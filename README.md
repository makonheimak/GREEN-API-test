# GREEN-API-test

Простой веб-клиент для проверки методов GREEN-API:
- `getSettings`
- `getStateInstance`
- `sendMessage`
- `sendFileByUrl`

## Структура проекта

- `index.html` — интерфейс приложения.
- `css/styles.css` — стили.
- `js/api.js` — класс `GreenAPI` и HTTP-запросы.
- `js/app.js` — основная логика UI (валидация, обработчики, рендер ответа).

## Быстрый старт

1. Откройте `index.html` в браузере.
2. Введите:
   - `idInstance`
   - `ApiTokenInstance`
3. Нажмите нужный метод и смотрите ответ в поле `Response`.

## Как пользоваться методами

### 1) getSettings / getStateInstance

Просто укажите `idInstance` и `ApiTokenInstance`, затем нажмите кнопку метода.

### 2) sendMessage

Заполните:
- `chatId` (например, `79991234567@c.us`)
- `message`

Нажмите `sendMessage`.

### 3) sendFileByUrl

Заполните:
- `chatId` (например, `79991234567@c.us`)
- `fileUrl` (публичная ссылка формата `https://...`)

Нажмите `sendFileByUrl`.

> Важно: `file:///C:/...` не подходит для `sendFileByUrl`, потому что сервер GREEN-API не имеет доступа к вашему локальному диску.

## Частые ошибки

### `idInstance and apiToken are required`

Причины:
- не заполнены поля `idInstance`/`ApiTokenInstance`;
- значения содержат лишние пробелы;
- используются неверные данные инстанса.

Проверьте корректность данных в личном кабинете GREEN-API.

### Ошибка при отправке файла

Обычно проблема в ссылке:
- ссылка должна быть доступна из интернета;
- лучше использовать прямую ссылку на файл (`https://.../file.pdf`), а не страницу с предпросмотром.

## Локальная разработка

Для удобства можно запускать через локальный HTTP-сервер (вместо `file://`), например:

```bash
python3 -m http.server 8080
```

После этого откройте в браузере:

`http://localhost:8080`
